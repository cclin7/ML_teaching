Parallel Computing in Python

Thread and process

Threads

Definition

  • Thread: A thread is the smallest unit of a process that can be scheduled for execution. It is sometimes referred to as a “lightweight process” because it runs within the context of a larger process and shares its resources (such as memory and file handles).

Key Points

  • Concurrency: Threads allow multiple parts of a program to run concurrently, sharing the same memory space.
  • Global Interpreter Lock (GIL): Python has a GIL which allows only one thread to execute at a time in a single process. This can be a bottleneck for CPU-bound tasks.
  • Use Cases: Threads are useful for I/O-bound tasks, such as reading/writing to files, network operations, and user interactions.

Example Code

import threading

def print_numbers():
    for i in range(10):
        print(i)

# Creating a thread
thread = threading.Thread(target=print_numbers)

# Starting the thread
thread.start()

# Waiting for the thread to finish
thread.join()

Processes

Definition

  • Process: A process is an instance of a program that is being executed. It contains its own memory space, file handles, and security context. Multiple processes can run independently and concurrently on a system.

Key Points

  • Isolation: Processes are isolated from each other. They do not share memory space, which prevents issues like race conditions but makes inter-process communication more complex.
  • Multiprocessing: Pythons multiprocessing module allows the creation of processes, bypassing the GIL limitation and taking full advantage of multiple CPU cores.
  • Use Cases: Processes are suitable for CPU-bound tasks that require heavy computation.

Example Code

import multiprocessing

def print_numbers():
    for i in range(10):
        print(i)

# Creating a process
process = multiprocessing.Process(target=print_numbers)

# Starting the process
process.start()

# Waiting for the process to finish
process.join()

Key Differences

  • Memory Space:
    • Threads share the same memory space.
    • Processes have separate memory spaces.
  • GIL:
    • Threads are limited by the GIL in Python.
    • Processes bypass the GIL and can run on multiple CPU cores.
  • Communication:
    • Threads communicate through shared memory.
    • Processes communicate using inter-process communication mechanisms like pipes and queues.

Summary

  • Thread: A lightweight unit of a process, sharing the same memory space. Suitable for I/O-bound tasks. Managed using the threading module.
  • Process: A heavier unit with its own memory space. Suitable for CPU-bound tasks. Managed using the multiprocessing module.

These concepts help in achieving concurrency and parallelism in Python programs, each suited for different types of tasks.

Multi-threads

Because of the GIL, multithreading is only meaningful in cases that involve a lot of I/O.

In Python, due to the Global Interpreter Lock (GIL), true multi-threading can be limited, especially for CPU-bound tasks. However, you can still use multiple threads for I/O-bound tasks. For CPU-bound tasks, you might want to use multiprocessing instead. Here are examples of both approaches:

The following provides an exmaple:

import requests
import time

URLS = [
    'https://www.example.com',
    'https://www.example.org',
    'https://www.example.net'
]

def fetch_content(url):
    response = requests.get(url)
    return response.text

start_time = time.time()

for url in URLS:
    fetch_content(url)

end_time = time.time()

print(f"Single-threaded took {end_time - start_time} seconds")
Single-threaded took 1.8319330215454102 seconds
import threading
import requests
import time

URLS = [
    'https://www.example.com',
    'https://www.example.org',
    'https://www.example.net'
]

def fetch_content(url):
    response = requests.get(url)
    return response.text

threads = []
start_time = time.time()

for url in URLS:
    thread = threading.Thread(target=fetch_content, args=(url,))
    threads.append(thread)
    thread.start()

for thread in threads:
    thread.join()

end_time = time.time()

print(f"Multi-threaded took {end_time - start_time} seconds")
Multi-threaded took 0.6622068881988525 seconds

a templet for multiprocessing

import multiprocessing
import time
import os

# Step 1: Define the complicated function
def complicated_function(data):
    # Simulate a complicated computation
    time.sleep(1)  # Replace with actual computation
    return f"Processed data: {data}"

# Step 2: Modify the wrapper function to load files
def wrapper(args):
    input1, input2 = args
    filename = f"file_{input1}_{input2}.txt"  # Construct the filename based on inputs
    try:
        with open(filename, 'r') as file:
            data = file.read()
        return complicated_function(data)
    except FileNotFoundError:
        return f"File {filename} not found."

if __name__ == "__main__":
    # Example inputs
    inputs1 = [1, 2, 3, 4]  # Example input set 1
    inputs2 = ['a', 'b', 'c', 'd']  # Example input set 2

    # Determine the number of available CPU cores
    total_cores = multiprocessing.cpu_count()
    print(f"Total available CPU cores: {total_cores}")

    # Set the number of processes to use (adjustable)
    num_processes = min(4, total_cores)  # For example, use up to 4 processes or the total number of cores, whichever is smaller
    print(f"Using {num_processes} CPU cores for parallel computation")

    # Create a list of argument pairs
    args_list = [(i1, i2) for i1 in inputs1 for i2 in inputs2]

    # Step 4: Use multiprocessing.Pool to parallelize
    pool = multiprocessing.Pool(processes=num_processes)
    results = pool.map(wrapper, args_list)

    # Close the pool and wait for the work to finish
    pool.close()
    pool.join()

    # Print the results
    for result in results:
        print(result)
